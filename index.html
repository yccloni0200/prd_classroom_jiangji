在线教室 · 分级降级策略

面向低端机型与弱网环境，将「一刀切式降级」升级为「L0–L3 可分级、可自适应、可自助」的性能降级体系，在保证教学闭环的前提下最大化课堂流畅度。

即上图后台「标准降级设备」配置：每个机型一个开关，命中即降级，不区分降级强度。

问题一 · 降级粒度太粗，力度不够
  • 只有「正常 / 降级」两态，且降级仅去掉固定 4 个模块。
  • 关闭这 4 项后，部分学生依旧明显卡顿（视频流、课件背景动画、音效、督学采帧等高耗模块仍在跑）。
  • 不同机型性能差异巨大，却用同一套降级配置，"够用"和"过度"并存。
问题二 · 用户无法自助，链路太长
  • 学生/家长卡顿时没有任何自主降级入口。
  • 只能：反馈 → 技术排查 → 后台加机型 → 下次生效。当堂课无法挽回。
  • 名单靠人工维护，新机型/个例覆盖滞后，客诉转化为流失风险。
本质：现有方案缺少「分级（强度可调）」与「自适应+自助（实时可控）」两个维度。本 PRD 围绕这两点重构降级体系。

2 目标与衡量

产品目标

  • 分级：建立 L0→L3 四档降级，强度逐级递增，按机型/实时性能精准匹配。
  • 自适应：客户端实时监测，卡顿自动升档、恢复自动回落。
  • 自助：教室内提供「流畅度」入口，用户可一键切到最流畅，无需等后台。
  • 守底线：核心教学链路（主课件、师生音频、答题）任何档位都不降。

北极星 & 核心指标

↓ 卡顿率
FPS<24 时长占比
↓ 卡顿客诉
卡顿类工单 / 千节课
↑ 完课率
监控降级是否伤害教学
↑ 自助使用
自助+自动降级覆盖率

辅助：进场耗时、掉线/重连次数、CPU/内存峰值、降级后卡顿改善率、续费/口碑（防过度降级）。

3 分级降级体系（L0–L3）

逐级累加降级：上一档关闭的能力，下一档继续保留关闭，再叠加更激进的项。降级 = 体验换流畅，核心教学能力始终保留。

L0 标准

全功能开启。性能良好的中高端设备 & 良好网络。

L1 轻度

关闭"锦上添花"动效，几乎无感，保留全部互动玩法。

L2 中度

关闭高耗互动+背景动画，视频转标清，明显省资源。

L3 极致流畅

纯净模式：仅保留课件+音频+必要互动，视频可降帧/关闭。

5 三层触发机制

三层来源共同决定最终档位，优先级：用户手动 > 客户端自适应 > 后台分级；后台可设「档位下限」防止过度降级。

🗄️
第 1 层 · 后台机型分级（默认档）
  • 现有「降级名单」升级为「机型 → 建议档位 + 档位下限」映射表。
  • 例：老旧 Android Pad → 默认 L2，下限 L1；iPad Air 2 → 默认 L1。
  • 白名单设备强制 L0；支持按版本灰度。进教室时下发为初始档。
＋
📈
第 2 层 · 客户端实时自适应（动态调档）
  • 持续采集 FPS、CPU/内存、丢帧、网络RTT/丢包。
  • 连续 N 秒低于阈值 → 自动升一档并轻提示；指标回升并稳定 → 自动回落（带滞回，避免抖动）。
  • 不会降到低于后台下限，也不会高于用户手动设定（除非用户选"自动"）。
＋
🙋
第 3 层 · 用户自助（最高优先）
  • 教室「⋯」菜单新增「流畅度」入口，4 档可选 + "自动"。
  • 选定后立即生效并记忆为该用户偏好（跨课生效，可在设置重置）。
  • 手动档位下，自适应不再自动降级，仅在更卡时提示"可进一步切换"。

🧮 档位仲裁器（可交互 · 拖动试试）

三个来源共同决定最终档位。拖动滑块，实时看仲裁结果：最终 = clamp( max(用户手动, 自适应建议), 后台下限, L0 )

🗄️ 后台档位下限

运营按机型配置的"最低也要降到这档"

📈 自适应建议

客户端按实时 FPS/CPU/网络给出

🙋 用户手动

教室内自助选择（0 = 未设 / 跟随自动）
最终生效档位
仲裁结果 = clamp(max(用户手动, 自适应建议), 后台下限, L0)。即：取"用户/自动"中更激进者，但不低于后台下限。

关键埋点

  • 档位变更事件（来源：后台/自动/手动 + 前后档位）
  • 实时性能采样（FPS/CPU/RTT 分布）
  • 自动降级触发率 / 撤销率
  • 自助入口曝光、点击、设置分布
  • 降级前后卡顿改善率、掉线率对比
  • 分档完课率、互动完成率（教学效果守护）

风险与兜底

  • 过度降级伤教学/续费 → 档位下限 + 核心链路不降 + 监控分档完课率。
  • 自动切换打断课堂 → 静默切换、转场时机、仅轻提示。
  • 降级抖动反复横跳 → 升降档滞回 + 冷却时间。
  • AI/Phototime 关闭破坏闭环 → 替代路径 / 可跳过。
  • 关视频影响师生互动 → L3 默认保最低帧，关闭需确认。

灰度与上线

  • 阶段一：后台分级名单替换黑名单（仅默认档，无自动/自助）。
  • 阶段二：开放用户自助入口，小流量灰度。
  • 阶段三：开启客户端自适应自动降级，按机型分组放量。
  • 每阶段以卡顿率↓、完课率不降为放量门槛。

9 待确认项（评审讨论）

  • L0–L3 各档具体关闭哪些能力是否需与教研确认（教学效果边界）？
  • 自适应阈值（FPS/N/CPU）初始值，需技术给出各机型基线数据。
  • 用户自助偏好是按账号还是按设备记忆？多设备如何同步？
  • 视频关闭涉及师生互动与监课合规，是否允许彻底关闭，还是仅最低帧？
  • 自助入口放在「⋯」菜单 vs 常驻按钮，哪个更易被低龄用户/家长发现？
